# Setting Up and Starting Jupyter Notebook (NVIDIA Internel)

This guide will help you set up and start Jupyter Notebook in your environment. There are two methods available: using our custom script or the ADLR script.

## Method 1: Using Our Custom Script

### Prerequisites

#### 1. Environment File Setup
First, you need to create a `.env` file in the `Eagle2` directory. For detailed instructions, please refer to [setting_env_file.md](./setting_env_file.md).

#### 2. Jupyter Configuration
Create a Jupyter configuration file at `/home/$USER/.jupyter/jupyter_notebook_config.py` with the following settings:

```python
# Allow connections from any IP address
c.NotebookApp.ip = '0.0.0.0'            
# Set the port number (default: 8888)
c.NotebookApp.port = 8888               
# Disable automatic browser opening
c.NotebookApp.open_browser = False      
# Optional: Set token for authentication
c.NotebookApp.token = ''                
# Optional: Set password for authentication
c.NotebookApp.password = ''             
```

> **Note**: If you want to secure your Jupyter Notebook with authentication, you can set either a token or password. Leave them empty if you don't need authentication.

### Starting Jupyter Notebook

To start Jupyter Notebook, run the following command:

```bash
bash tools/nv_cluster_tools/start_jupyter.sh
```

### Port Forwarding Setup

Since users outside the US cannot access the internal server IP directly, our script implements port forwarding from the server to the management node (e.g., forwarding port 8888 from the server to port 9107 on the management node).

To access Jupyter Notebook from your local machine:

1. First, ensure the port forwarding is set up on the management node
2. Then, set up port forwarding on your local machine (easily done through VSCode)
3. Access Jupyter Notebook at `localhost:9107` (assuming you forwarded the management node's port 9107 to your local machine)

> **Note**: The port numbers (9107 in this example) may vary based on your specific setup. Please check with your system administrator for the correct port numbers.


## Method 2: Using ADLR Script for US users.

The ADLR scripts provide an alternative way to start Jupyter Notebook. While this method works well for users within the US network, users outside the US may experience connectivity issues due to network restrictions. To address this, we implement port forwarding to access the internal server.

```bash
run_jupyter --partition cpu_long --gpu 0 --cpu 96 --duration 168 --image $Path/to/docker/image
```
